Dokku探索历程-1
2019-01-02|技术总结容器技术|

很多年前偶然机会接触过Heroku，对于开发人员来说这个PaaS平台很友好，可以通过配置文件和Git直接部署应用而不需要关心Infra级别的操作。

现在的Heroku有Dyno容器的概念，会对不在运行的程序进行回收休眠操作，并且当时注册Heroku的学生奖金也已经用完了，这让我萌生了寻找私有化服务的想法。

如果以有一台闲置的VPS或者主机（比如EC2或者阿里云），可以选择安装Dokku提供类似Heroku一般的服务。

常用应用安装

Dokku 安装完成后，可以通过执行命令

查看dokku是否安装成功并且查看各种命令的帮助手册。

Commands块描述了本身Dokku携带的一些功能。包括：

  1. Application管理 (Application Management)
  2. Application Monitoring & Debugging，包括如何查看容器运行程序的日志，指定计划任务的执行周期，执行指定命令等。
  3. Network and SSL，查看和指定Application对应开放的端口，配置Nginx转发策略和子域名，查看和启停用SSL证书等。
  4. Git and SSH，配置Git和添加信任密匙等。
  5. Storage，关联宿主机和容器程序的存储对应管理。
  6. Dokku本身的相关命令，如和Docker关联的相关配置关系，以及是否Troubleshoot程序本身的功能。
